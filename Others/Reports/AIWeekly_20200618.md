#### Is artificial intelligence changing art?
- https://scienceline.org/2020/06/is-artificial-intelligence-changing-art/
- 과연 인공지능이 새로운 "예술"을 창조할 수 있을까? 에 대한 토론
- 부정적 의견
  - 의견: 지금 인공지능이 하는 방식은 과거에도 있었던 방식의 효율화일 뿐이다
    - 과거에도 무작위로 단어를 뽑고 조합하는 방식이 존재 => 현재 인공지능이 만드는 문학 작품과 유사한 방식이며 속도가 빨라졌을뿐임
  - 의견: 인공지능은 결국 인간이 인공지능을 "멋지다" 라고 여기는 형태를 만들고 있을뿐이고 이는 과거에도 있는 것들이다
  - 의견: 인공지능의 예술은 사실상 "기술 전시회" 로 여겨진다
- 긍정적 의견
  - 의견: 사람들이 "기술 전시회" 라고 여긴다고 해서 그것이 창의적이지 않은 것은 아니다
  - 의견: 기술 자체도 하나의 "예술" 이 될수 있다
    - 영화, 소설, 그림 등은 모두 다른 형태이지만 동일하게 "예술"로 불린다

#### AI Research, Replicability and Incentives
- https://dennybritz.com/blog/ai-replication-incentives/
- AI 연구의 떨어지는 재현성을 막기위해서는 성과 보장 방식 자체의 변화가 필요하다는 주장
- ML에 대한 연구 자체는 꾸준히 증가하지만 재현 가능한 연구가 점점 줄고있음
  - 동일한 환경에서 동일한 실험을 해도 동일한 결과가 나오지 않는 연구 증가
- 왜 이런 결과가 생기는가?
  - 프레임워크의 차이
  - 미묘한 구현 차이
    - 논문으로만 발표된 연구에서 특히
  - 랜덤 시드에 따른 큰 결과 차이
  - 하이퍼 파라미터에 따라 큰 결과 차이
  - 특정 데이터 셋에서만 동작하는 모델
  - 최적화 방법별 성능 차이
  - 평가 방식의 차이
  - 테스트셋의 유출 (의도적이건 그렇지 않건)
  - 데이터 파이프라인 및 전처리 방법 차이
  - 버그
- 왜 이런게 유지되는가: 재현성을 유지하는것보다 SOTA를 따라잡는게 더 큰 성과로 취급
- 어떻게 이를 해결할 수 있을까?
  - 재현에 대한 표준을 만들수 있지만 이런 방식은 새로운 시도를 막아버릴 위험이 있음
  - 중요한건 화제가 되지 않는 논문이라도 성과에 대한 보상이 주어져야 함
    - Top-down 방식 성과 보장
      - 큰 규모의 집단에 적합
      - 평가자가 새로 가치 창출이 가능한 연구를 발견하면 이를 다양하게 도입 가능하도록 루트를 만들어주어야 함
      - 이 과정에서 위험성이 있더라도 이에 대한 감수가 가능한 시스템 도입 필요
    - Bottom-Up 방식 성과 보장
      - 작은 집단내에서 가치 창출이 가능한 연구 발견시 이를 큰 규모의 조직에 전달할 수 있는 루트 필요
      - 기존 학계의 관습이나 고착된 문화를 회피 가능
      - 대표적인 예시가 Github 의 도입
        - 대부분의 회사들이 자체적인 솔루션을 도입하였지만 말단 조직에서 Github을 사용한 결과 현재는 대부분의 회사 단위로 Github 도입



#### Learning local and compositional representations for zero-shot learning
- https://www.microsoft.com/en-us/research/blog/learning-local-and-compositional-representations-for-zero-shot-learning/
- 피쳐의 locality와 이 피쳐들을 조합하기 위한 compositionality 를 강조하는 방식이 zero-shot 연구에서 높은 성능을 보여줌
- zero-shot
  - 학습 단계에서 데이터가 주어지지 않아도 예측을 할 수 있는 모델
  - ex) Horse를 학습시키고 Stripes를 학습시킨뒤 입력으로 Horse + Stripes를 넣어주면 얼룩말 출력
- 이미지에서 중요한 파트가 어딘지 학습할 수 있도록 모델을 여러 모듈로 나누고, 각각 모듈에 별개로 학습을 수행
  - 전체 역전파 사용하지 않음
- 이후 Tree Reconstruction Error을 최소화 하는 형태로 모듈간의 조합을 학습시킴
- Transfer-learning 은 제로샷에서 성능에 도움이 되지 않음
- 결과를 End-To-End 방식과 비교
  - 비지도 학습들에 비해서는 모두 더 높은 성능 보여줌
  - 지도 학습 방식에 비해서도 특정 상황에선 더 높은 성능 나타남
